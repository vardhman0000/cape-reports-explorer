# 🔍 CAPEv2 Reports - JSON Explorer

A lightweight, zero-dependency local web app for exploring large JSON files beautifully in your browser. Built for analysts working with thousands of structured reports (malware, logs, pipelines, etc.).

No npm. No pip installs. Just Python.

---

## Background

Reports are generated by the [CAPEv2](https://github.com/kevoreilly/CAPEv2) malware sandbox.
JSON Explorer was built to make browsing those large analysis reports faster and easier locally.

---

## Versions

### v1 — Single File Viewer (`json_viewer.py`)
View and explore a single JSON file with a rich interactive UI.

```bash
python json_viewer.py report_1266.json
python json_viewer.py report_1266.json --port 8080
```

### v2 — Directory Browser (`json_explorer_v2.py`) ⭐ Recommended
Browse an entire directory of `report_*.json` files. Click any report to load and inspect it — all without leaving the page.

```bash
# Looks for ./analyses/ by default
python json_explorer_v2.py

# Point to a custom directory
python json_explorer_v2.py --dir /path/to/analyses
python json_explorer_v2.py --dir ./analyses --port 8080
```

## Features

### File Browser (v2)
- Lists all `.json` files in your directory with file sizes
- Filter reports by number or name instantly
- Sort by Report ID, Name, or File Size
- Lazy loading — files are only fetched when clicked, so startup is instant even with thousands of reports

### JSON Tree Viewer
- Syntax-highlighted collapsible tree (keys, strings, numbers, booleans, nulls each get distinct colors)
- Large nodes (25+ items) start collapsed for performance
- Long strings are truncated with a click-to-expand toggle
- Lazy rendering — child nodes are built only when expanded

### In-Report Search
- `Ctrl+F` focuses the search bar
- All matches highlighted inline, current match highlighted in gold
- `↑` / `↓` buttons (or keyboard) to step through matches
- Live match counter (e.g. `4/27`)

### Other
- **Expand All / Collapse All** buttons
- **Stats overview** — file size, key counts, type distribution breakdown
- **Copy Path** button — copies the dot-path of the current section to clipboard
- Auto-opens the browser on start

## Requirements

- Python 3.7+
- No third-party packages — uses only the standard library (`http.server`, `json`, `pathlib`, `argparse`, `webbrowser`)

## Directory Structure

```
json-explorer/
├── json_viewer.py          # v1 — single file viewer
├── json_explorer_v2.py     # v2 — directory browser
├── README.md
└── analyses/               # your report_*.json files go here (not committed)
```

---

## Usage with a Custom Directory Name

Your report directory doesn't have to be named `analyses`. Pass any path:

```bash
python json_explorer_v2.py --dir ./reports
python json_explorer_v2.py --dir ~/data/cape_reports --port 9000
```
